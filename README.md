# Low-Level Network Socket Application

Веб-додаток, побудований виключно на стандартній бібліотеці Python, що демонструє низькорівневу мережеву взаємодію через HTTP та UDP протоколи. HTTP-сервер відповідає за маршрутизацію та роздачу статичних ресурсів, тоді як окремий Socket-сервер обробляє та зберігає дані форми — обидва працюють паралельно у різних потоках.

---

## Технологічний стек

| Технологія     | Призначення                                        |
|----------------|----------------------------------------------------|
| Python 3.13    | Основна мова програмування                         |
| `http.server`  | Вбудований HTTP-сервер                             |
| `socket`       | Низькорівневий сокет-сервер для обробки повідомлень|
| `threading`    | Паралельний запуск обох серверів                   |
| `json`         | Постійне зберігання повідомлень                    |
| Docker         | Контейнеризація додатку                            |
| Bootstrap 5    | Стилізація фронтенду                               |

## Архітектура

```
Браузер
   │
   │  HTTP (порт 3000)
   ▼
┌──────────────┐        UDP пакет          ┌───────────────────┐
│  HTTP Сервер │ ──────────────────────>   │  Socket Сервер    │
│  (порт 3000) │                           │  (порт 5000)      │
└──────────────┘                           └───────────────────┘
       │                                           │
       │  Роздає HTML-сторінки                     │  Парсить та зберігає
       │  та статичні файли                        ▼
       ▼                                   storage/data.json
  index.html
  message.html
  error.html
```

**Як це працює:**
1. HTTP-сервер обробляє GET-запити (маршрутизація, статичні файли) та POST-запити (відправка форми)
2. При відправці форми HTTP-сервер пересилає сирі дані на Socket-сервер через UDP
3. Socket-сервер парсить дані, додає мітку часу та зберігає їх у `storage/data.json`
4. Обидва сервери запускаються в окремих потоках з єдиної точки входу `main.py`

## Запуск проєкту

### Варіант 1: Запуск через Docker (рекомендовано)

```bash
# Клонуємо репозиторій
git clone https://github.com/Khrystyna979/low-level-net-app.git
cd low-level-net-app

# Збираємо та запускаємо контейнер
docker compose up --build
```

Додаток буде доступний за адресою **http://localhost:3000**

Повідомлення зберігаються у `./storage/data.json` на локальній машині завдяки Docker volume — дані не зникають після зупинки контейнера.

### Варіант 2: Локальний запуск

```bash
cd low-level-net-app
python main.py
```

## Сторінки та маршрутизація

| Маршрут     | Метод | Опис                                                       |
|-------------|-------|------------------------------------------------------------|
| `/`         | GET   | Головна сторінка з оглядом курсу Python                    |
| `/message`  | GET   | Сторінка з формою для відправки повідомлення               |
| `/message`  | POST  | Відправка форми -> пересилання через UDP -> редирект на `/`  |
| `/style.css`| GET   | Статичний CSS-файл                                         |
| `/logo.png` | GET   | Статичний файл зображення                                  |
| `*`         | GET   | Будь-який невідомий маршрут → `error.html` зі статусом 404 |

## Формат зберігання даних

Кожне надіслане повідомлення зберігається у `storage/data.json` з міткою часу як ключем:

```json
{
    "2026-06-22 13:06:25.881503": {
        "username": "John",
        "message": "Hello, World!"
    },
    "2026-06-22 13:06:41.236956": {
        "username": "Anna",
        "message": "Hi!"
    }
}
```

## Ключові технічні деталі

- **Без сторонніх фреймворків** — HTTP-сервер побудований повністю на модулі `http.server` зі стандартної бібліотеки
- **UDP протокол** — легковагна, без'єднувальна комунікація між HTTP та Socket серверами
- **`threading.Thread`** — обидва сервери запускаються в окремих потоках в рамках одного процесу
- **`mimetypes`** — автоматично визначає `Content-Type` для статичних файлів
- **`urllib.parse`** — декодує URL-encoded дані форми (`application/x-www-form-urlencoded`)
- **Автоініціалізація** — директорія `storage/` та файл `data.json` створюються автоматично при запуску, якщо вони відсутні
